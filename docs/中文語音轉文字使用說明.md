# 中文語音轉文字使用說明

把要轉文字的音檔放進 `audio-inbox/`，再從 repo 根目錄執行 `transcribe-zh.cmd`。

## 基本用法

```powershell
.\transcribe-zh.cmd ".\audio-inbox\sample.m4a"
```

也可以使用絕對路徑：

```powershell
.\transcribe-zh.cmd "D:\path\to\sample.m4a"
```

輸出會放在：

```text
transcripts/
```

輸出檔案包含：

- `.txt`：純文字逐字稿
- `.srt`：字幕檔
- `.16k.wav`：Whisper 使用的暫存轉檔音訊

## 腳本位置

主要腳本在：

```text
scripts\transcribe-zh.ps1
```

根目錄的 `transcribe-zh.cmd` 是方便執行的入口。

## 目前設定

- 引擎：`whisper.cpp`
- 預設模型：`models\ggml-base.bin`
- 預設語言：`zh`
- 預設模式：CPU，使用 `--no-gpu`
- 預設提示詞：繁體中文演講逐字稿

這是目前在這台電腦上測到比較穩的設定。之前不加 `--no-gpu` 時，完整音檔容易變成無法解釋的亂碼。

## 實測速度

測試音檔：

```text
D:\User\文件\Playground\one\voice\sample\20250420練講三寶.m4a
```

結果：

- 音訊長度：約 47 分 30 秒
- 轉錄耗時：約 27 分 22 秒
- 推估速度：1 小時音訊約 35 分鐘

## 目前輸出範例

完整測試結果已集中在測試資料夾：

```text
transcripts\test-runs\full_nogpu_prompt\20250420____.txt
transcripts\test-runs\full_nogpu_prompt\20250420____.srt
```

注意：`base` 模型速度還可以，但後段可能出現重複文字或聽錯字。若之後要提高準確度，可以改用 `small` 或 `medium` 模型。

測試過程產生的短音檔、失敗輸出和中繼檔也都放在：

```text
transcripts\test-runs\
```

## 之後請 Codex 幫你轉

你可以直接把音檔放到：

```text
audio-inbox\
```

然後跟我說：

```text
幫我轉 audio-inbox 裡的音檔
```

我會用這個流程執行，並把 `.txt` 和 `.srt` 放到 `transcripts/`。
