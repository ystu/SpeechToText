# 中文語音轉文字使用說明

這個專案已整理成本機離線轉錄流程。之後要轉中文錄音時，把音檔放進 `audio-inbox`，再執行 `transcribe-zh.cmd`。

## 常用方式

在 `D:\User\文件\Playground\meeting` 執行：

```powershell
.\transcribe-zh.cmd ".\audio-inbox\你的音檔.m4a"
```

也可以直接指定其他資料夾的音檔：

```powershell
.\transcribe-zh.cmd "D:\User\文件\Playground\one\voice\sample\20250420練講三寶.m4a"
```

輸出會放在：

```text
transcripts\
```

每次會產生：

- `.txt`：純文字逐字稿
- `.srt`：字幕檔
- `.16k.wav`：Whisper 使用的中繼音訊

## 目前使用的方法

目前預設使用：

- `whisper.cpp`
- `ggml-base.bin`
- 中文模式：`zh`
- CPU 模式：`--no-gpu`
- 預設提示詞：繁體中文演講逐字稿

這是目前在這台電腦上測到比較穩的設定。之前不加 `--no-gpu` 時，完整音檔容易變成「無法解釋」或亂碼。

## 實測速度

測試音檔：

```text
D:\User\文件\Playground\one\voice\sample\20250420練講三寶.m4a
```

結果：

- 音訊長度：約 47 分 30 秒
- 轉錄耗時：約 27 分 22 秒
- 推估速度：1 小時音訊約 35 分鐘

## 目前輸出範例

完整測試結果已集中在測試資料夾：

```text
transcripts\test-runs\full_nogpu_prompt\20250420____.txt
transcripts\test-runs\full_nogpu_prompt\20250420____.srt
```

注意：`base` 模型速度還可以，但後段可能出現重複文字或聽錯字。若之後要提高準確度，可以改用 `small` 或 `medium` 模型。

測試過程產生的短音檔、失敗輸出和中繼檔也都放在：

```text
transcripts\test-runs\
```
